고령자·구음장애 한국어 STT 서비스형 MVP를 기준으로, 현재 실제로 동작하는 범위와 AI-Hub 기반 학습 파이프라인의 준비 상태를 정리한 문서입니다.

샘플 세그먼트 96개
학습 전략 LoRA Encoder
현재 단계 MVP + 학습 준비

프로젝트 인포그래픽

현재 구현된 기능

서비스/웹

  • 홈, 로그인, 회원가입, 업로드, 작업 조회, 결과 상세, 관리자 화면 구현
  • 파일 업로드와 브라우저 녹음 지원
  • SSE 기반 상태 스트리밍과 작업 진행률 표시
  • 문장별 timestamp, confidence, low-confidence 강조 표시
  • 사용자 결과 수정 및 correction 저장

백엔드/API

  • JWT 인증, 업로드, 작업 생성/상태 조회/결과 저장 API
  • FastAPI + PostgreSQL + Redis + Celery + MinIO 연동
  • 관리자 통계, 모델 비교, correction export API
  • 오디오 재생과 결과 비교를 위한 관리자 상세 조회

STT 추론

  • 16kHz mono 변환, VAD, Whisper 계열 추론 래퍼
  • segment timestamp / confidence 계산
  • 후처리와 low-confidence 구간 식별 구조
  • mock 모드와 실제 로컬 모델 모드 분리

학습 파이프라인

  • AI-Hub 샘플 데이터 구조 자동 인식
  • 긴 자유대화 wav를 학습용 세그먼트로 분할
  • train/valid/test manifest 생성
  • LoRA / encoder-only / full 학습 전략 골격
  • CSV correction export 로딩 지원

리팩토링 결과와 샘플 검증

학습 파이프라인은 더 이상 단순 예시 코드가 아니라, 실제 AI-Hub 샘플 데이터를 받아 세그먼트 생성과 manifest 분할까지 수행하는 구조로 정리됐습니다. 다만 현재 PC에는 학습 의존성이 아직 설치되지 않아 실제 fine-tuning 실행은 남아 있습니다.

아직 구현해야 할 기능

실제 학습 실행

  • `torch`, `transformers`, `datasets`, `peft`, `soundfile` 설치
  • 샘플 데이터 기준 1차 Whisper fine-tuning 실행
  • WER/CER 평가 결과 저장

데이터 정교화

  • forced alignment 또는 WhisperX/VAD 기반 정렬 도입
  • AI-Hub 전체 데이터셋 포맷 추가 대응
  • correction 데이터와 AI-Hub 데이터 통합 재학습셋 구성

운영 배포 자동화

  • CTranslate2 변환 및 faster-whisper 연결
  • `model_versions` 자동 등록과 운영 승격 배치
  • 모델별 품질 비교 자동화

서비스 품질 고도화

  • 테스트 확대, CI/CD, 모니터링, 알림 체계
  • 보안 강화와 역할 기반 권한 분리
  • 공공/돌봄 현장 대응을 위한 운영 정책 정비

1단계: 학습 실행

현재 준비된 manifest와 trainer를 기준으로 샘플 fine-tuning과 1차 WER/CER 측정을 진행합니다.

2단계: 데이터 품질 향상

정렬 기반 세그먼트 보정과 correction 데이터 통합으로 실제 현장 발화 적응력을 높입니다.

3단계: 운영 모델 승격

학습 결과를 faster-whisper 운영 추론 구조에 연결하고 모델 버전 관리 흐름을 완성합니다.

4단계: 서비스 안정화

배포 자동화, 모니터링, 보안, 테스트를 보강해 공공/돌봄 환경 운영 수준으로 끌어올립니다.

참고: 현재 프로젝트는 "서비스형 MVP + 학습 파이프라인 준비 완료" 단계이며, 실제 AI-Hub 기반 fine-tuning 실행은 학습 의존성 설치와 적절한 실행 환경 확보 이후 이어서 진행해야 합니다.
